ğŸ” What Is Unit Testing?

Unit tests check that each small part of your code (a function or method) works correctly. If one part breaks, your test will catch it before your users do.

ğŸ§° Why You Should Care

  • - Avoid bugs before they sneak into your project
  • - Understand your code better
  • - Save time debugging later
  • - Works great with GitHub and CI tools

âš™ï¸ Built-in Testing with unittest

# my_math.py
def add(a, b):
    return a + b

# test_my_math.py
import unittest
from my_math import add

class TestAdd(unittest.TestCase):
    def test_add_positive(self):
        self.assertEqual(add(2, 3), 5)

    def test_add_negative(self):
        self.assertEqual(add(-1, -1), -2)

if __name__ == '__main__':
    unittest.main()

Run it from your terminal like this:

python test_my_math.py

Testing with pytest is much easier!

If you want simpler syntax, use pytest:

# test_my_math.py
from my_math import add

def test_add_positive():
    assert add(2, 3) == 5

def test_add_negative():
    assert add(-1, -1) == -2

To run:

pip install pytest  
pytest

Tips for Beginners

  • - Write tests as you build functions
  • - Store test files like test_*.py
  • - Use GitHub Actions or CI tools to run tests automatically
